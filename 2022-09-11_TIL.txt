협업을 위한 Git 활용 기초 3주차

최신 commit 고치기 - amend
- 최신 커밋을 amend 로 수정
- 강제 push 할 경우에 다른 사람에게 지장을 줄 수 있으니 신중하게 해야한다.

commit 되돌리기 revert, reset
- revert는 변경 기록을 남기면서 커밋을 되돌리는 방식
- reset 을 아예 삭제하는 방식으로 soft, mixed, hard 가 있다.
- hard는 작업내역을 아예 삭제해서 없애서 복원할수 없다
- mixed는 커밋은 되돌리고 작업 내용은 보존해서 변경사항으로 보여준다. 변경사항은 add 된 상태로 보인다.
- soft는 커밋은 되돌리고 작업 내용은 보존해서 변경사항으로 보여준다. 변경상태는 add 상태 되지 않은 상태로 보인다.

stash
- 변경 사항을 임시로 보관

커밋 메시지
- 컨벤션이 지켜진 커밋 메시지를 통해 보다 원활한 협업을 지향

코드 리뷰
- 다른이가 내 코드를 보고 리뷰 해주는 것
- 대체로 깃에서 PR 한 내역에서 댓글을 달면서 리뷰를 남기는 방식을 사용

gitignore
- 깃에서 제외할 내용을 정해주는 것

README
- 안내글

오픈소스
- 공개되어 있는 코드를 바탕으로 진행되는 프로젝트
- 누구나 프로젝트에 참여할 수 있으며 집단 지성으로 발전
